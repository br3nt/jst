1. Mutable references can be republished explicitly

Recommended pattern

Direct assignment to a mutable prop is treated as a publish signal, even when the array/object reference is unchanged. Parent `.prop` bindings still skip unchanged references so parent renders do not fan out through the whole tree.

2. Form ownership: local model vs parent-owned events

    Design check
    
This is an API design check, not a bug demo. `jst-model` updates local component props. Parent-owned form state stays explicit with `.value` and an emitted event, so ownership is visible at the call site. Vue and Alpine lean into model sugar; React keeps controlled inputs explicit.

3. Event handlers close over render-time values

    Problem / footgun
    
Bare props are plain JS values captured when the template rendered. When the next state depends on the previous state, read from the live element property, then assign back to that property.

6. Keyed transitions preserve identity, but are not full FLIP

    Capability limit
    
This checks two things: keyed nodes should keep their live DOM state while reordering, and `jst-transition` should stay a small CSS class-phase helper. Complex layout choreography still belongs in custom JS or a helper outside core.

7. Side effects should usually live outside templates

    Recommended pattern
    
I agree this logic usually does not belong in a template. The preferred shape is page-owned side effects and dumb renderers. `onDisconnect(fn)` should stay an escape hatch for DOM-local resources a component truly creates itself, not an invitation to put business logic in templates.
