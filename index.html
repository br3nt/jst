What it is

JST turns a <script type="jst"> tag into a real custom element. You write components in ordinary HTML, with JavaScript as the templating language — no compiler, no bundler, no JSX, no virtual DOM. Drop one <script> on a page and you have reactive, composable components.

  </button>

The hole it fills

JST started from a question: what would a front-end SSR component look like? Could the server send down components, like HTMX — but with real client-side interactivity, like Alpine — without a build step, a virtual DOM, or signals?

ToolStrengthThe gap JST targets
HTMXServer returns HTML; hypermedia-driven.No client-side interactivity without a round-trip; introduces its own attribute language.
AlpineSprinkle client interactivity into HTML.No server-streamed components; ad-hoc scopes and stores.
React / VuePowerful component models.Build step, virtual DOM, framework runtime, JSX/SFC tooling.
JSTComponents in plain HTML that the server can stream down (a fetched fragment can define and use a component — it auto-registers), with full client interactivity, no build, and no virtual DOM or signals — it rides the platform: custom elements, properties, events, DOM morphing.

We tested that claim: 70 canonical examples from HTMX, Alpine, Vue and React, rebuilt in JST. 35 were exact matches, 35 needed a documented workaround, none were impossible. The gaps are catalogued honestly.

Philosophy

  • No build step. Components live in <script type="jst"> in plain HTML. The browser is the toolchain.
  • JavaScript is the templating language. $(expr) to interpolate, $ if / $ forEach for control flow. No new DSL to learn.
  • Ride the platform. Real custom elements. Rich data flows as properties; actions flow out as bubbling CustomEvents. The DOM is patched in place by morphing — no virtual DOM, no signals.
  • Props down, events up. Components are dumb renderers; state lives in the page or a parent, recursively. Easy to reason about, easy to test.
  • The server can ship components. A fetched HTML fragment can carry a brand-new component definition and markup using it; JST registers it on arrival. HATEOAS, with a component model.
  • Safe by default. Interpolation is HTML-escaped unless you opt out with raw().
